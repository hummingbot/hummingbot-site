## Release Highlights

### 🏛️ Architect Perpetual Connector

We are excited to introduce the new **Architect** connector, integrating [Architect Exchange](https://architect.exchange) into the Hummingbot ecosystem. This release delivers a full perpetual futures connector so algorithmic traders can automate strategies against Architect APIs, including a sandbox environment for safer testing.

**Highlights**

* **Perpetual Futures:** Supports order placement, cancellation, fills, and position lifecycle for perpetual markets (one-way position mode with token-specific leverage; align leverage with venue rules when configuring strategies).
* **Sandbox & QA:** Exchange sandbox is available for end-to-end testing of strategies before mainnet use.
* **Community Bounty:** Developed and delivered through a Hummingbot community bounty — see issue [#7919](https://github.com/hummingbot/hummingbot/issues/7919).
* **Core Integration:** Officially merged into the Hummingbot client via PR [#7951](https://github.com/hummingbot/hummingbot/pull/7951).

**Thanks to [petioptrv](https://github.com/petioptrv) for this contribution! 🙏**

### ⚡ Evedex Perpetual Connector

Hummingbot now includes a connector for **Evedex** perpetual derivatives, extending automated trading to Evedex hybrid perpetuals. The integration wires up configuration, rate sources, REST and WebSocket flows—including user-stream handling built on Centrifugo—for a complete perpetual trading path in the client.

**Highlights**

* **Perpetual Derivatives:** Automate perpetual strategies with integrated market data and user streams suited to low-latency workflows.
* **Rate Sources:** Registers Evedex and Evedex Perpetual rate source modes so pricing can participate in the broader Hummingbot rate-oracle ecosystem.
* **Real-Time User Data:** User stream support covers WebSocket connectivity, subscriptions, keepalive, and message processing over Centrifugo.
* **Core Integration:** Officially merged into the Hummingbot client via PR [#7995](https://github.com/hummingbot/hummingbot/pull/7995).

**Thanks to [isreallee82](https://github.com/isreallee82) for this contribution! 🙏**

### 🔊 Decibel Perpetual Connector

This release adds a **Decibel** perpetual connector, bringing Hummingbot to Decibel trading on [mainnet](https://app.decibel.trade/) and [testnet](https://testnet-decibel.vercel.app/). API access is provided through [Geomi](https://geomi.dev/), with integration built on Decibel [Python SDK](https://github.com/decibeltrade/python-sdk) so strategies can run perpetual market making and related workflows with the same client UX as other perpetual venues.

**Highlights**

* **Mainnet & Testnet:** Trade and test against both production and testnet deployments from a single connector design.
* **Perpetual Workflows:** Supports perpetual market making with orders, cancellations, fills, and position tracking (as validated by the contributor on mainnet and testnet).
* **Community Bounty:** Developed and integrated following completion of the community bounty in issue [#8028](https://github.com/hummingbot/hummingbot/issues/8028).
* **Core Integration:** Officially merged into the Hummingbot client via PR [#8098](https://github.com/hummingbot/hummingbot/pull/8098).

**Thanks to [dizpers](https://github.com/dizpers) for this contribution! 🙏**

### 🎯 GRVT Perpetual Connector

We have added a new connector for **GRVT**, a hybrid derivatives exchange focused on perpetual futures with a self-custodial trading model. Hummingbot users can now connect the standard client to GRVT perpetual markets to deploy and manage automated strategies alongside other supported venues.

**Highlights**

* **Perpetual Automation:** Place, update, and manage perpetual orders and positions through the familiar Hummingbot derivative connector model.
* **Native Client Support:** Configure and operate GRVT perpetual markets without leaving the core Hummingbot client workflows.
* **Community Bounty:** Developed and delivered through the GRVT-sponsored community bounty — see issue [#8046](https://github.com/hummingbot/hummingbot/issues/8046).
* **Core Integration:** Officially merged into the Hummingbot client via PR [#8107](https://github.com/hummingbot/hummingbot/pull/8107).

**Thanks to [gtwototoo](https://github.com/gtwototoo) for this contribution! 🙏**

### 📊 Strategy V2 Backtesting Improvements

This release significantly upgrades the **Strategy V2** backtesting framework, adding richer executor simulation, clearer position and fee accounting, and optional interactive visualization so you can validate controller and executor strategies on historical OHLCV data with more realistic PnL reporting.

**Highlights**

* **Grid Executor Simulator:** Simulates grid-style executors on historical candles—including configurable grid levels, fills inferred from candle high/low, take-profit recycling, per-row performance metrics, and global controls such as time limits and aggregate PnL-based stops.
* **Position Holds, Fees & PnL Series:** Introduces `BacktestPositionHold` to accumulate and net positions when executors keep exposure open, with separate handling for executor vs. position PnL to avoid double-counting; applies consistent round-trip fee accounting (entry and exit) across position, DCA, and grid simulators; adds tick-level PnL timeseries with breakdowns for executor realized, position realized, and unrealized components.
* **Results & Visualization:** Extends `BacktestingResult` with structured summary output and optional Plotly charts (candles with executor entry/exit markers, cumulative PnL components, and position held over time); install `plotly` to generate interactive figures.
* **Core Integration:** Merged via PR [#8177](https://github.com/hummingbot/hummingbot/pull/8177).

### 💱 Rate Oracle Refactor & Binance Perpetual WebSocket

This release refactors **fee and PnL calculation for perpetuals** so quote conversions flow through the **Rate Oracle** instead of threading exchange instances through orders, fees, and `MarketsRecorder`. The Rate Oracle is wired up with your connectors at initialization and can resolve **inverse trading pairs** when a direct quote is unavailable—yielding more consistent fee and PnL reporting across spot and perpetual workflows, including Strategy V2 executors. The same change set includes an **upgrade fix for the Binance perpetual user WebSocket** (so streams stay healthy after the upstream WS changes) and **serialization fixes** for persisted trade history uncovered during the refactor.

**Highlights**

* **Oracle-Centric Fees & PnL:** Routes quote conversions for perpetual fees and PnL through the Rate Oracle (removing exchange arguments from orders, fees, and `MarketsRecorder`); initializes the oracle with active connectors and resolves **inverse pairs** when a direct quote path is missing.
* **Strategy V2 & Trade History:** Updates Strategy V2 executors to the new API and fixes `MarketsRecorder` serialization so orders, trades, and fees persist and reload correctly from SQLite across restarts.
* **Binance Perpetual WebSocket:** Updates the Binance perpetual user stream for compatibility with upstream WebSocket changes so user stream connections and order updates remain stable during longer runs.
* **Core Integration:** Merged via PR [#8149](https://github.com/hummingbot/hummingbot/pull/8149).
