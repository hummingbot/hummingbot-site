## Release Highlights

### ü§ñ Condor ‚Äî Telegram Bot for Managing Hummingbot Instances

**Condor** is a new **Telegram-based control center** for Hummingbot, giving traders a fast and intuitive way to monitor bots, run trades, manage liquidity, and interact with Gateway ‚Äî all directly from their phone.

Built on the Hummingbot API, Condor provides a streamlined interface for both CEX and DEX operations, offering real-time dashboards, interactive menus, and a unified configuration system. Whether you're swapping tokens, managing CLMM liquidity, or monitoring your grid strategies, Condor brings full Hummingbot control into a lightweight, mobile-friendly UI.

**Key Features**

* **Portfolio Dashboard:** View PnL, asset distribution, positions, LP states, and historical charts 
* **Bot Management:** Monitor multiple bots, controllers, metrics, logs, and status in real time
* **CEX & DEX Trading:** Place spot/perp orders and execute DEX swaps with quotes and slippage control
* **Liquidity Management:** Browse pools, manage CLMM positions, collect fees, and close positions
* **Server & API Key Configuration:** Add/edit/delete Hummingbot API servers and exchange credentials 
* **AI Assistant (Coming Soon):** Natural-language trading through GPT-4o + MCP tooling

Condor makes Hummingbot more accessible than ever ‚Äî perfect for traders running bots across multiple environments who want lightweight, remote monitoring and execution without needing a desktop terminal. The [Hummingbot API Quickstart](../installation/hummingbot-api.md) includes step-by-step Condor setup instructions. See [Condor](../condor/index.md) for more information.

### üõ†Ô∏è Gateway Connector & Infrastructure Enhancements

This release delivers a wide set of improvements across Gateway‚Äôs Solana and EVM connectors, focusing on **token reliability**, **pool discovery**, **error-handling consistency**, and **CLMM stability**. Together, these updates strengthen cross-chain LP workflows, improve swap accuracy, and make Gateway more resilient to real-world RPC/provider quirks.

**Highlights**

* **Dynamic & Robust Token Handling** [#543](https://github.com/hummingbot/gateway/pull/543)
  Gateway now supports **unlisted ERC-20 tokens**, dynamic wrapped-token detection, and graceful fallbacks when external RPCs (like Helius) return mixed parsed/base64 formats. This resolves issues in Polygon wrapping, BigInt conversions, and post-transaction balance extraction.

* **CLMM Stability Improvements (Meteora, Raydium, Orca, PancakeSwap-Sol)** [#550](https://github.com/hummingbot/gateway/pull/550)
  Added fixes for **InvalidPositionWidth** errors, accurate Meteora fee reporting, improved user-facing swap/allowance errors, and reliable handling of unknown pool tokens. CLMM operations across Solana ecosystems now behave consistently and predictably.

* **WebSocket Monitoring & Real-Time Discovery Tools** [#555](https://github.com/hummingbot/gateway/pull/555)
  Introduced **Helius and Infura WebSocket listeners** for live transaction monitoring, plus **CoinGecko-based token & pool discovery** with automatic caching. This enhances ecosystem coverage and allows instant visibility into new tokens and pool opportunities.

* **Standardized Error Handling Framework** [#563](https://github.com/hummingbot/gateway/pull/563) [#564](https://github.com/hummingbot/gateway/pull/564)
  A new centralized `HttpError` service standardizes error messages and HTTP responses across ~70 connector routes, improving clarity and debugging consistency for all Solana and EVM connectors.

* **Expanded DEX & Provider Compatibility** [#569](https://github.com/hummingbot/gateway/pull/569)
  Added GeckoTerminal DEX mappings for Uniswap V3 on major EVM networks, upgraded Meteora SDK, improved PancakeSwap-Sol parser resilience, migrated Jupiter to its new API, and unified all external service keys (Helius, Infura, CoinGecko, Etherscan) under a shared config model. 


### üîê Hyperliquid API Key Authentication Now Supported

Hummingbot now supports **API key‚Äìbased authentication** for both the **Hyperliquid Spot** and **Hyperliquid Perpetual** connectors, offering a more flexible alternative to the existing Arbitrum wallet + private key setup.

With this enhancement, traders can generate API key/secret credentials directly from the **Hyperliquid web interface** and use them to place orders, manage positions, and run automated strategies through Hummingbot.

**Highlights**

* **Dual Authentication Modes:** Choose between API key authentication or Arbitrum wallet signing, depending on your workflow and security preferences.
* **Spot + Perpetual Coverage:** Unified API key support across both `hyperliquid` (spot) and `hyperliquid_perpetual` (perp) connectors.
* **Simplified Setup:** Configure your API credentials directly in Hummingbot without needing an on-chain private key.

This update streamlines the onboarding experience for Hyperliquid users and enhances compatibility with a broader range of trading setups.

üìö Documentation: [Hyperliquid Connector Guide](../exchanges/hyperliquid.md)

**Special thanks to [gtwototoo](https://github.com/gtwototoo) for this contribution! üôè**
